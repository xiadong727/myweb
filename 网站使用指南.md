# 拾光共长 - 网站使用与维护详细指南

这份文档是你个人网站的“说明书”。你的网站是一个**静态站点**，所有的内容更新都在你当前的 **Cursor 编辑器** 中通过修改本地文件来完成。修改完成后，一键同步到 GitHub，网站就会自动更新。

---

## 核心揭秘：左侧目录树是如何生成的？

网站左侧的“多级树状目录”完全由 `data/navigation.json` 这个文件控制。你可以把它想象成网站的“骨架”。

### 1. 哪里指定分类名称和文章名称？

*   **分类名称（目录名）**：在 `data/navigation.json` 中，每个分类节点大括号里的 `"title"` 字段。
*   **左侧菜单显示的文章名称**：在 `data/navigation.json` 中，文章节点大括号里的 `"title"` 字段。
*   **文章页面正文里显示的大标题**：在 `.md` 文章文件最开头的 `title: "..."` 里面。（强烈建议：导航栏的 title 和文章里的 title 保持一致）。

### 2. 一级、二级、三级目录分类是在哪里指定的？

在 `data/navigation.json` 中，层级是通过**嵌套 `"children"`** 来实现的。
*   带有 `"children"` 的项，就是一个**分类（文件夹）**，它可以被折叠和展开。
*   带有 `"slug"` 的项，就是一篇**具体的文章（文件）**，点击它会跳转到具体页面。
*   把一个分类放到另一个分类的 `"children"` 里面，就形成了多级目录。

#### 详解示例：如何创建一、二、三级目录？

打开 `data/navigation.json`，找到 `"articles": { "nodes": [ ... ] }`。所有的文章分类都在这个 `nodes` 数组里。下面是一个包含三级目录的完整结构示例：

```json
"nodes": [
  {
    "id": "folder-level-1",
    "title": "这是一级分类（例如：技术笔记）",
    "children": [
      {
        "id": "folder-level-2",
        "title": "这是二级分类（例如：前端开发）",
        "children": [
          {
            "id": "folder-level-3",
            "title": "这是三级分类（例如：React教程）",
            "children": [
              {
                "id": "article-01",
                "title": "这是具体的文章名称",
                "slug": "tech/frontend/react-intro"
              }
            ]
          },
          {
            "id": "article-02",
            "title": "这是一篇直接放在二级分类下的文章",
            "slug": "tech/frontend/other-article"
          }
        ]
      }
    ]
  }
]
```

**关键字段解释（必看）：**
1.  **`id`**: 每个大括号里都必须有一个 `id`。这个名字你自己随便起（最好是英文），只要整个文件里**不重复**就行。系统用它来区分不同的节点。
2.  **`title`**: 网页上实际显示的中文名字。
3.  **`children`**: 只要你想让这个节点变成一个“可以展开的文件夹”，就必须加 `"children": []`。
4.  **`slug`**: 只有具体的文章/作品才有 `slug`。它对应着你在 `content/articles/` 文件夹下创建的 `.md` 文件的路径（不需要写 `.md` 后缀）。

## 实战操作指南

### 一、 如何增加一篇新文章？

发文章的核心逻辑是**“两步走”**：先写文章文件，再去导航栏登记。

**第一步：写内容**
1. 在 `content/articles/` 文件夹下，新建一个 `.md` 文件（最好用英文/拼音命名，比如 `my-diary.md`）。你可以自由地在里面创建真实文件夹来整理文件。
2. 打开文件，在最开头必须写上“头部信息”（Frontmatter），格式如下：
```markdown
---
title: "今天天气真好"
date: "2026-04-21"
description: "这是文章的简短摘要，会显示在列表里。"
---

这里开始写正文...
```

**第二步：挂到左侧菜单**
1. 打开 `data/navigation.json`。
2. 找到你想放入的分类的 `"children"` 数组里，增加一段代码：
```json
{
  "id": "leaf-my-diary",
  "title": "今天天气真好",
  "slug": "my-diary" 
}
```
*注意：`slug` 必须是你刚才创建的文件的名字（不带 .md）。如果你把它放在了 `notes` 文件夹下，slug 就是 `notes/my-diary`。记得在上一项的末尾加逗号 `,`。*

### 二、 如何修改、删除文章？

*   **修改文章内容**：直接打开 `content/articles/` 里的 `.md` 文件修改保存。
*   **修改文章在菜单上的名字**：打开 `data/navigation.json`，修改对应节点的 `"title"`。
*   **删除文章**：
    1. 在 `content/articles/` 里把那个 `.md` 文件删掉。
    2. 打开 `data/navigation.json`，把对应的那段 `{ "id": "...", "title": "...", "slug": "..." }` 代码删掉。

### 三、 如何管理图片/视频/音频？

图片、视频、音频的管理逻辑是**“一步走”**：直接修改对应的数据文件即可。

**以增加图片为例：**
1. 把你的图片文件（比如 `cat.jpg`）拖进 `public/images/` 文件夹。
2. 打开 `data/galleries.json`。
3. 找到对应的分类（比如 `"daily"`），在它的 `"items"` 数组里增加一段：
```json
{
  "id": "pic-cat-01",
  "title": "我家的小猫",
  "description": "今天抓拍到它在睡觉。",
  "url": "/images/cat.jpg",
  "date": "2026-04-21"
}
```

**以增加视频/音频为例：**
1. 强烈建议把视频/音频上传到 B站、网易云或云存储，获取一个播放链接（URL）。
2. 打开 `data/videos.json` 或 `data/audios.json`。
3. 在对应的分类下增加一段：
```json
{
  "id": "vid-01",
  "title": "我的年度总结",
  "description": "2025年的精彩瞬间",
  "url": "https://www.bilibili.com/video/BVxxxxxx",
  "coverUrl": "/images/video-cover.jpg",
  "date": "2026-04-21"
}
```

### 四、 如何在文章中插入图片？

在文章（Markdown）中插入图片有两种方式，取决于你想把图片文件放在哪里：

**方式一：图片和文章放在一起（推荐）**
适合只给这篇（或这个分类的）文章用的图片。
1. 在 `content/articles/` 里，和你的 `.md` 文件同级的目录下建一个 `images/` 文件夹。
2. 把图片放进去。
3. 在 Markdown 里这样写（不要加最前面的斜杠）：
   ```markdown
   ![](images/你的图片名.png)
   ```

**方式二：使用全站公用的图片**
适合多篇文章都会用到的通用图片（比如 Logo）。
1. 把图片放到项目根目录的 `public/images/` 文件夹下。
2. 在 Markdown 里这样写（必须以 `/` 开头）：
   ```markdown
   ![](/images/你的图片名.png)
   ```

### 五、 如何调整文章的排版样式（字体大小、行距等）？

网站的文章排版是统一管理的。如果你觉得字太小、行距太挤，可以修改 `src/components/article-body.tsx` 文件。

打开该文件，找到 `components={{ ... }}` 这部分代码，里面定义了所有 Markdown 标签的样式（使用的是 Tailwind CSS 语法）：

*   **修改正文（`p` 标签）**：找 `p: (props) => ...`
    *   `text-[1.05rem]`：控制字体大小
    *   `leading-[1.8]`：控制行间距（1.8倍行距）
    *   `tracking-[0.02em]`：控制字间距
    *   `style={{ textIndent: "2em" }}`：控制首行缩进两格
    *   `text-justify`：控制两端对齐
*   **修改标题（`h1`, `h2`, `h3`）**：找对应的标签，修改 `text-[...]` 调整大小，修改 `mt-...` 调整距离上一段的空白。
*   **修改列表（`ul`, `ol`）**：找对应的标签，修改 `space-y-...` 调整列表项之间的距离。

每次修改保存后，网页会自动刷新显示最新排版。

## 最后一步：如何发布到网上？（同步到 GitHub）

当你在本地做完了所有的增加、删除、修改，并且按 `Ctrl+S` (或 `Cmd+S`) 保存了文件后，你需要把它们推送到网上。

**只需三步：**
1. **打开源代码管理**：点击 Cursor 最左侧边栏的第三个图标（长得像树枝分叉的图标，叫 Source Control）。
2. **写更新说明**：在顶部的输入框（Message）里写一句话，比如：“新增了一篇日记和两张猫咪图片”。
3. **提交并推送**：
   * 点击输入框下方的 **Commit（提交）** 按钮。
   * 提交完成后，按钮会变成 **Sync Changes（同步更改）**，再点击它一下。

代码同步到 GitHub 后，Vercel 会在后台全自动为你更新网站。你只需要等待 1~2 分钟，刷新你的手机或电脑浏览器，就能看到最新内容了！
