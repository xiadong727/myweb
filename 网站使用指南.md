# 拾光共长 - 网站使用与维护详细指南

这份文档是你个人网站的“说明书”。你的网站是一个**静态站点**，所有的内容更新都在你当前的 **Cursor 编辑器** 中通过修改本地文件来完成。修改完成后，一键同步到 GitHub，网站就会自动更新。

> 如果你要更新的是「与光同行」主线的某一期内容，**直接看下面这一节就够了**，它是为这件主线大事专门写的傻瓜流程。其它零散作品（随笔、摄影、视频）再看后面的通用操作指南。

---

## 与光同行｜每期发布 SOP（主线项目专用）

这是主线要长期重复的核心动作。把每一期当成一件**作品**来做，但用固定流程降低每次的决策成本。整个流程分四步：**写稿 → 命名归位 → 挂到目录 → 预览发布**。

### 0. 先认识「与光同行」的目录结构

所有主线内容都放在 `content/articles/lighthouse/` 下（`lighthouse` 是内部英文路径名，菜单上显示的是「与光同行」，不用改它）。文件夹按 **领域** 分成若干个：

| 文件夹 | 领域（菜单显示） | 文件夹 | 领域（菜单显示） |
| --- | --- | --- | --- |
| `L01-philosophy` | 哲学 (L01) | `L06-art` | 艺术 (L06) |
| `L02-thinking` | 思维 (L02) | `L07-relationship` | 人际 (L07) |
| `L03-character` | 品格 (L03) | `L08-wealth` | 财富 (L08) |
| `L04-history` | 历史 (L04) | `L09-life` | 生活 (L09) |
| `L05-science` | 科学 (L05) | `L10-spirit` | 心灵 (L10) |

每写一期，先想清楚：**这期属于哪个领域？** 然后就把文件放进对应的那个文件夹。

### 1. 写稿：用固定的文章结构模板

每期正文建议按这 6 段来写（这是文档《战略方案》5.6 节定下的结构，能保证“既有深度，又能讲给孩子听”）：

1. **引子（约500字）**：一个故事/场景，让大人和孩子都能进入。
2. **正文（1500–2500字）**：核心智慧的拆解，至少 3 个层次——是什么 / 为什么 / 怎么用。
3. **古今实例（约500字）**：1 个历史例子 + 1 个当代例子，证明这个智慧穿越周期。
4. **讲给孩子听（300–500字）**：用一个故事或比喻，把道理讲给 7 岁孩子。
5. **行动建议（约200字）**：本周可以做的 1–3 件小事。
6. **延伸阅读**：1–3 本书 + 相关文献。

### 2. 命名归位：把文件放进对应领域文件夹

1. 进到对应领域文件夹，比如哲学就是 `content/articles/lighthouse/L01-philosophy/`。
2. 新建一个 `.md` 文件，**文件名统一用 `epNNN-英文简称` 的格式**（N 是 3 位期号，全程递增、永不重复）。例如第 7 期讲“知行合一”，就叫 `ep007-knowing-and-doing.md`。
3. 在文件最开头，**原样复制下面这段头部信息（Frontmatter）**，再按你这一期的实际情况改值：

```markdown
---
title: "本期标题"
date: "2026-01-12"
excerpt: "一句话摘要，会显示在列表和搜索结果里。"
series: "lighthouse"
episode: 7              # ← 第几期（同领域内唯一，决定该领域内的排序）
domain: "L01"          # ← 所属领域编号（L01–L10）
quote: "本期核心金句"
forKids: "用一句话写给孩子听的版本"
books: ["《推荐书1》", "《推荐书2》"]
score: 13              # ← 选题自检评分，≥12 才发（见下方自检表）
media:
  audio: false         # ← 这期有没有配套音频
  video: false         # ← 这期有没有配套视频
access: "free"         # ← free 免费 / member 会员专属（以后开会员用）
---

# 本期标题

这里开始写正文……
```

> **为什么要老老实实填这些字段？** 现在多填几行几乎不费事，但将来想做“按期号排的时间线”“10 领域进度看板”“年度内容体检”，靠的全是这些字段，到时不用回头补几百篇。

> **💡 填了 `quote` 会自动生成「金句卡」**：只要这篇文章 frontmatter 里有 `quote`，文章正文末尾就会出现一个金句区块和「保存金句卡」按钮，点一下就能下载一张 1080×1350 的竖版分享图（暖色品牌风），直接发朋友圈/小红书。不填 `quote` 就不显示，互不影响。

**插图**：和文章放一起最省心。在该 `.md` 同级建一个 `images/` 文件夹（L01 已经有了），把图片丢进去，正文里写 `![](images/图片名.png)`（**不要**加最前面的斜杠）。

### 3. 挂到左侧目录（不挂就看不到）

写好文件后，网页上还不会出现，必须去 `data/navigation.json` 登记一下。

1. 打开 `data/navigation.json`，找到最上面的 `"id": "lighthouse"` 这个大节点。
2. 找到这一期所属领域的小节点，比如哲学是 `"id": "lh-L01"`。
3. 在它的 `"children": []` 里加一条（空的领域第一次加要把 `[]` 改成 `[ ... ]`）：

```json
{
  "id": "lh-L01-ep007",
  "title": "第007期 · 知行合一",
  "slug": "lighthouse/L01-philosophy/ep007-knowing-and-doing"
}
```

> 三个要点：① `id` 全文件不能重复，建议就用 `lh-领域-ep期号`；② `title` 是左侧菜单显示的字，建议带上“第NNN期”；③ `slug` 必须等于文件相对 `content/articles/` 的路径（不带 `.md`），一个字都不能错。记得在上一条末尾补逗号。

### 4.（可选）一鱼三吃：同一期配上音频/视频

文档 4.1 说一篇好文章要“一鱼三吃”：本体只写 1 次，再拆成音频、视频。网站会**自动把同一期的文章、音频、视频、图片互相链接**——在文章底部出现「🐟 一鱼三吃 · 本期其它形式」卡片，点一下就能跳过去。

**关联靠的是一个「期号键」**：文章的期号键 = `domain` + 期号，自动算出来。比如这期 `domain: "L01"`、`episode: 7`，期号键就是 **`L01-ep007`**（期号补足 3 位）。只要给配套音/视频/图片也写上同一个键，它们就会自动关联起来：

- 音频：在 `data/audios.json` 加一条，加一个字段 `"episode": "L01-ep007"`；再到 `navigation.json` 的「音频」树里挂好。
- 视频：在 `data/videos.json` 加一条（推荐 B站/云存储的 embed 链接），同样加 `"episode": "L01-ep007"`；再挂到「视频」树。
- 图片：在 `data/galleries.json` 加一条，同样加 `"episode": "L01-ep007"`。
- frontmatter 里的 `media: { audio: true, video: true }` 是给你自己做记录看的标记（可填可不填），**真正决定互链的是 `episode` 键**，不用它也能关联。

（音频/视频/图片的详细字段写法，见后面《如何管理图片/视频/音频》一节。）

### 5. 预览与发布

1. 终端运行 `npm run dev`，打开 `http://localhost:3000/articles`，展开“与光同行”→ 对应领域，确认这一期能点开、排版正常、图片显示出来。
2. 没问题就按最后一节《如何发布到网上》提交并同步到 GitHub。

### 发布前的每期自检清单（贴在心里）

发之前快速过一遍（来自《战略方案》附录B），**总分 < 12★ 的选题先放回去打磨，不要硬发**：

- [ ] 这期属于哪个领域，我心里清楚（frontmatter 的 `domain` 填对了）？
- [ ] 我有没有真的读过至少 1 本相关原典/主书？
- [ ] 这期有没有“我自己/我和孩子”的真实亲历成分？
- [ ] 用一句话能讲清这期的核心吗（就是 `quote`）？
- [ ] 「讲给孩子听」那段，7 岁孩子真能听懂吗？
- [ ] `episode` 期号有没有和别的期撞号？
- [ ] 选题自检评分 `score` ≥ 12 了吗？

### 一句话速记

> 选好领域 → 在 `lighthouse/对应领域/` 建 `epNNN-xxx.md` 并填好 frontmatter → 去 `navigation.json` 的 `lh-领域` 节点挂一条 → `npm run dev` 预览 → 同步 GitHub。

## 核心揭秘：左侧目录树是如何生成的？

网站左侧的“多级树状目录”完全由 `data/navigation.json` 这个文件控制。你可以把它想象成网站的“骨架”。

### 1. 哪里指定分类名称和文章名称？

*   **分类名称（目录名）**：在 `data/navigation.json` 中，每个分类节点大括号里的 `"title"` 字段。
*   **左侧菜单显示的文章名称**：在 `data/navigation.json` 中，文章节点大括号里的 `"title"` 字段。
*   **文章页面正文里显示的大标题**：在 `.md` 文章文件最开头的 `title: "..."` 里面。（强烈建议：导航栏的 title 和文章里的 title 保持一致）。

### 2. 一级、二级、三级目录分类是在哪里指定的？

在 `data/navigation.json` 中，层级是通过**嵌套 `"children"`** 来实现的。
*   带有 `"children"` 的项，就是一个**分类（文件夹）**，它可以被折叠和展开。
*   带有 `"slug"` 的项，就是一篇**具体的文章（文件）**，点击它会跳转到具体页面。
*   把一个分类放到另一个分类的 `"children"` 里面，就形成了多级目录。

> **展开/折叠规则**：每次进入或刷新页面时，左侧目录**默认只展开第一级分类，所有子目录都是折叠的**。需要看子分类时，点一下分类名（前面的小箭头）即可展开，再点一下收起。这样目录再深也不会一进来就刷屏。

#### 详解示例：如何创建一、二、三级目录？

打开 `data/navigation.json`，找到 `"articles": { "nodes": [ ... ] }`。所有的文章分类都在这个 `nodes` 数组里。下面是一个包含三级目录的完整结构示例：

```json
"nodes": [
  {
    "id": "folder-level-1",
    "title": "这是一级分类（例如：技术笔记）",
    "children": [
      {
        "id": "folder-level-2",
        "title": "这是二级分类（例如：前端开发）",
        "children": [
          {
            "id": "folder-level-3",
            "title": "这是三级分类（例如：React教程）",
            "children": [
              {
                "id": "article-01",
                "title": "这是具体的文章名称",
                "slug": "tech/frontend/react-intro"
              }
            ]
          },
          {
            "id": "article-02",
            "title": "这是一篇直接放在二级分类下的文章",
            "slug": "tech/frontend/other-article"
          }
        ]
      }
    ]
  }
]
```

**关键字段解释（必看）：**
1.  **`id`**: 每个大括号里都必须有一个 `id`。这个名字你自己随便起（最好是英文），只要整个文件里**不重复**就行。系统用它来区分不同的节点。
2.  **`title`**: 网页上实际显示的中文名字。
3.  **`children`**: 只要你想让这个节点变成一个“可以展开的文件夹”，就必须加 `"children": []`。
4.  **`slug`**: 只有具体的文章/作品才有 `slug`。它对应着你在 `content/articles/` 文件夹下创建的 `.md` 文件的路径（不需要写 `.md` 后缀）。

## 实战操作指南

### 一、 如何增加一篇新文章？

发文章的核心逻辑是**“两步走”**：先写文章文件，再去导航栏登记。

**第一步：写内容**
1. 在 `content/articles/` 文件夹下，新建一个 `.md` 文件（最好用英文/拼音命名，比如 `my-diary.md`）。你可以自由地在里面创建真实文件夹来整理文件。
2. 打开文件，在最开头必须写上“头部信息”（Frontmatter），格式如下：
```markdown
---
title: "今天天气真好"
date: "2026-04-21"
excerpt: "这是文章的简短摘要，会显示在列表和搜索结果里。"
---

这里开始写正文...
```

**第二步：挂到左侧菜单**
1. 打开 `data/navigation.json`。
2. 找到你想放入的分类的 `"children"` 数组里，增加一段代码：
```json
{
  "id": "leaf-my-diary",
  "title": "今天天气真好",
  "slug": "my-diary" 
}
```
*注意：`slug` 必须是你刚才创建的文件的名字（不带 .md）。如果你把它放在了 `notes` 文件夹下，slug 就是 `notes/my-diary`。记得在上一项的末尾加逗号 `,`。*

### 二、 如何修改、删除文章？

*   **修改文章内容**：直接打开 `content/articles/` 里的 `.md` 文件修改保存。
*   **修改文章在菜单上的名字**：打开 `data/navigation.json`，修改对应节点的 `"title"`。
*   **删除文章**：
    1. 在 `content/articles/` 里把那个 `.md` 文件删掉。
    2. 打开 `data/navigation.json`，把对应的那段 `{ "id": "...", "title": "...", "slug": "..." }` 代码删掉。

### 三、 如何管理图片/视频/音频？

#### 先搞懂：每个板块的目录结构

文章、图片、视频、音频四个板块，左侧目录现在是同一套结构——**先是主线「与光同行」，后面是你自己的作品**：

| 板块 | 主线：与光同行（配套） | 你的作品（独立） |
| --- | --- | --- |
| 文章 | 与光同行 → 哲学(L01)…心灵(L10) | 日常·随笔 / 亲子·教育 / 创业·搞钱 / 工具·方法 |
| 图片 | 与光同行 · 金句卡 | 个人作品 ／ 作品学习（人文地理·旅行 · 光影与季节 · 生活记录 · 素材库） |
| 视频 | 与光同行 · 每期视频 → L01…L10 | 个人作品 ／ 作品学习（纪录片 · 短片·Vlog · 分享·演讲 · 作品集） |
| 音频 | 与光同行 · 每期音频 → L01…L10 | 个人作品 ／ 作品学习（播客 · 有声书 · 音乐·环境音） |

> **「个人作品」和「作品学习」怎么分？** 图片/视频/音频三个板块各有这两个分类：**个人作品**放你自己原创的成品；**作品学习**放你临摹、拆解、学习他人作品时的练习与笔记。两者是并列的同级分类，按内容性质往里放就行。

> 现在每个板块都留了**一个标着「（示例·可删）」的样板**（图片"作品学习→光影与季节→配色与光感"、视频"作品学习→分享·演讲→频道介绍"、音频"作品学习→播客→第一期：寻找心流"）。你照着它改最快，等你有真内容了就把样板删掉。

#### 通用逻辑：永远是「两步走」

图片、视频、音频都一样：**第一步在数据文件里配置资源，第二步去 `navigation.json` 挂到菜单**。两步用 `slug` 对上号——数据文件里的 `slug` 和菜单里的 `slug` 必须**一模一样**。

**第一步：配置资源数据**

🖼️ **图片相册** —— 编辑 `data/galleries.json`
1. 把图片拖进 `public/images/` 文件夹（比如 `cat.jpg`）。
2. 在 `galleries.json` 里找到或新建一个相册对象，往 `"images"` 数组里加图：
```json
{
  "slug": "photography/my-cat",
  "title": "我家的猫",
  "description": "2026 春天",
  "cover": "/images/cat.jpg",
  "images": [
    { "src": "/images/cat.jpg", "alt": "晒太阳的猫" }
  ]
}
```

🎬 **视频** —— 编辑 `data/videos.json`（强烈建议传到 B站/云存储拿播放链接）
```json
{
  "slug": "works/my-video",
  "title": "我的年度总结",
  "description": "2026 的精彩瞬间",
  "kind": "embed",
  "embedUrl": "https://player.bilibili.com/player.html?..."
}
```

🎧 **音频** —— 编辑 `data/audios.json`
```json
{
  "slug": "podcasts/ep2",
  "title": "第二期：如何陪孩子读书",
  "description": "一次睡前对话",
  "src": "https://你的音频直链.mp3",
  "cover": "https://封面图链接.jpg"
}
```

**第二步：挂到左侧菜单（非常重要，不挂就看不到）**

打开 `data/navigation.json`，找到对应板块（`"images"` / `"videos"` / `"audios"`），在你想要的分类 `"children"` 里加一条菜单项：
```json
{
  "id": "vid-works-myvideo",
  "title": "我的年度总结",
  "slug": "works/my-video"
}
```
*三个要点：① `slug` 必须和第一步数据文件里的 `slug` 完全一致；② `id` 全文件唯一，建议带板块前缀（图片 `img-`、视频 `vid-`、音频 `aud-`）；③ 记得在上一条末尾补逗号。*

#### 如果是「与光同行」某一期的配套音/视频（一鱼三吃）

这时要放进主线那一支，并和文章共用「期号键」，网站就会自动互链（文章底部出现「本期其它形式」卡片）：
- **给这条数据加一个 `episode` 字段**，值 = 文章的 `domain` + 期号，例如第7期哲学就是 `"episode": "L01-ep007"`（期号补 3 位）。
- 数据照上面写进 `audios.json` / `videos.json` / `galleries.json`，slug 仍按你自己的习惯命名即可。
- 挂菜单时挂到「与光同行 · 每期音频/视频」下对应领域节点（音频 `aud-lh-L01`、视频 `vid-lh-L01`……）。

```json
{
  "slug": "lighthouse/L01/ep007-audio",
  "title": "第007期 · 知行合一（音频）",
  "description": "本期音频版",
  "src": "https://你的音频直链.mp3",
  "episode": "L01-ep007"
}
```
> 只要文章（`domain:"L01"` + `episode:7`）和这条音频都对应 `L01-ep007`，两边详情页就会自动出现互相跳转的卡片。视频、图片同理。

### 四、 如何在文章中插入图片？

在文章（Markdown）中插入图片有两种方式，取决于你想把图片文件放在哪里：

**方式一：图片和文章放在一起（推荐）**
适合只给这篇（或这个分类的）文章用的图片。
1. 在 `content/articles/` 里，和你的 `.md` 文件同级的目录下建一个 `images/` 文件夹。
2. 把图片放进去。
3. 在 Markdown 里这样写（不要加最前面的斜杠）：
   ```markdown
   ![](images/你的图片名.png)
   ```

**方式二：使用全站公用的图片**
适合多篇文章都会用到的通用图片（比如 Logo）。
1. 把图片放到项目根目录的 `public/images/` 文件夹下。
2. 在 Markdown 里这样写（必须以 `/` 开头）：
   ```markdown
   ![](/images/你的图片名.png)
   ```

### 五、 如何调整文章的排版样式（字体大小、行距等）？

网站的文章排版是统一管理的。如果你觉得字太小、行距太挤，可以修改 `src/components/article-body.tsx` 文件。

打开该文件，找到 `components={{ ... }}` 这部分代码，里面定义了所有 Markdown 标签的样式（使用的是 Tailwind CSS 语法）：

*   **修改正文（`p` 标签）**：找 `p: (props) => ...`
    *   `text-[1.05rem]`：控制字体大小
    *   `leading-[1.8]`：控制行间距（1.8倍行距）
    *   `tracking-[0.02em]`：控制字间距
    *   `style={{ textIndent: "2em" }}`：控制首行缩进两格
    *   `text-justify`：控制两端对齐
*   **修改标题（`h1`, `h2`, `h3`）**：找对应的标签，修改 `text-[...]` 调整大小，修改 `mt-...` 调整距离上一段的空白。
*   **修改列表（`ul`, `ol`）**：找对应的标签，修改 `space-y-...` 调整列表项之间的距离。

每次修改保存后，网页会自动刷新显示最新排版。

### 六、 如何在本地预览修改效果？

在把内容发布到网上之前，你可以启动**本地开发服务器**，实时查看你的修改效果（这叫做热更新）。

**操作步骤：**
1. **打开终端**：在 Cursor 中，按下快捷键 `Cmd + J` (Mac) 或 `Ctrl + J` (Windows) 打开底部的终端面板。
2. **启动服务**：在终端里输入以下命令并回车：
   ```bash
   npm run dev
   ```
3. **预览网页**：在你的电脑浏览器（Chrome、Safari 等）的地址栏中输入 `http://localhost:3000` 并打开。

**💡 实时预览的神奇之处：**
启动服务后，你**不需要**反复执行这个命令。你可以在 Cursor 里随意修改文章内容或配置文件。每次你按下 `Cmd + S` / `Ctrl + S` 保存文件时，浏览器里的网页就会**瞬间自动刷新**，显示你最新的修改结果！这非常适合一边写文章一边看排版效果。

**如何停止预览？**
当你看完效果不想再运行服务时，点击 Cursor 终端面板，按下快捷键 `Ctrl + C` 就可以停止服务器了。

### 七、 网站地址设置（搜索引擎 / RSS 订阅，只需做一次）

网站自带了三样东西，帮助别人找到你、订阅你：
- `sitemap.xml`（站点地图，给百度/谷歌等搜索引擎看的全站目录）
- `robots.txt`（告诉搜索引擎哪些能收录）
- `feed.xml`（RSS 订阅源，老读者可以用阅读器订阅你的更新）

它们里面要用到你网站的**真实网址**。默认是占位地址 `https://example.com`，**部署到你自己的域名后**需要设置一次：

1. 打开 [Vercel](https://vercel.com) → 进入你的项目 → **Settings** → **Environment Variables**。
2. 新增一条：名字填 `NEXT_PUBLIC_SITE_URL`，值填你的真实网址（例如 `https://yourdomain.com`，**结尾不要带斜杠**）。
3. 保存后，到 **Deployments** 里点最近一次部署的 **Redeploy** 重新部署一次即可生效。

> 还没绑定自己的域名也没关系，先不用管这一步，等有了正式域名再设置。

### 八、 这些是网站自带的、不用你操心的功能

下面这些都是自动的，你正常写文章就行，不需要额外配置：
- **阅读进度条**：文章页顶部有一条细线，随滚动显示读到哪了。
- **文章目录（TOC）**：正文里有 2 个以上小标题（`##`、`###`）时，文章开头自动生成可点击的目录，滚动时高亮当前章节。
- **预计阅读时长**：根据字数自动估算，显示在标题下方。
- **上一篇 / 下一篇**：文章底部自动出现。灯塔系列按期号顺序（上一期/下一期），其它文章按发布日期。
- **空目录占位**：左侧还没放内容的分类，展开后会显示「敬请期待 · 内容建设中…」，不会是一片空白。
- **友好的 404 页**：访客点到不存在的链接时，会看到一个带「回到首页 / 浏览文章」的引导页面，而不是冰冷的报错。
- **标签聚合页**：文章 frontmatter 里写了 `tags` 后，文章页的标签可点击，跳到 `/tags/标签名` 看该主题下的全部文章；`/tags` 是所有标签的总览。
- **图片灯箱**：图集里的图片点一下会全屏放大，支持左右切换、Esc 关闭。
- **文章分享配图**：把文章链接发到微信/朋友圈/X 时，会自动带一张品牌化预览大图（标题 + 「拾光共长 · 与光同行」，暖色卡片）。无需你操作。

### 九、 首页都有哪些板块（全自动，随内容生长）

首页（`src/app/(site)/page.tsx`）从上到下依次是这些板块，全部**自动生成**。你平时只管照前面的流程写文章、填好 frontmatter，首页就会自己更新，**不用手动维护**：

1. **顶部数据面板**：右侧四个大数字会**自动统计**并以翻滚动画呈现——
   - **总作品数**：文章 + 图片 + 视频 + 音频 的总数；
   - **总浏览量**：所有作品浏览量之和（依赖数据库，见第十一节；没开通时显示 0）；
   - 下方分项：**文章 / 图片 / 视频 / 音频** 各自的数量。
   加内容就自动 +1，无需手动改。
2. **四个入口卡片（文章/图片/视频/音频）**：右上角带数量徽章，点击进入对应板块。
3. **与光同行 · 10 大领域进度墙**：把 10 个领域排成一面墙，自动数出每个领域**已更新几期**（靠文章 frontmatter 里的 `domain` 字段统计），有内容的格子点进去是该领域**最新一期**，没内容的显示「敬请期待」。顶部「已更新 N 期」也是自动算的。
   - 想让某个领域亮起来、计入期数，关键就是文章里 `domain: "L0X"` 要填对（见前面《每期发布 SOP》）。
4. **最新更新**：自动列出**最近 6 篇有 `date` 的文章**，按日期从新到旧排。所以记得每篇都填 `date`，不填的不会出现在这里。
5. **本期金句**：自动取**最新一篇带 `quote` 的文章**，把金句放大居中展示，点一下跳到原文。所以你想换首页展示哪句，就给最新那篇文章填好 `quote` 即可。

> **想改「了解这件事」按钮指向的文章？** 进度墙右上角那个按钮，默认链接到项目介绍文章。它的目标地址写在 `page.tsx` 顶部的 `const ABOUT_SLUG = "cogrow/10years02";` 这一行——把引号里的 slug 换成你想指向的那篇文章路径即可（路径写法同 `navigation.json` 里的 `slug`）。

**首页进度墙的标题和「进入主线」按钮，会跳到主线总览页 `/lighthouse`**——这是「与光同行」的项目主页，长这样（也全自动，不用维护）：

- 顶部是**概览**：已发布期数 + 已开启领域数（只数数量，不设期限、不显示年份，适合你不定时更新的节奏）。
- **最近更新**：按发布日期取最近 5 期，回访的人一眼看到新内容。
- **按领域浏览**：每个领域一段，段内按期号列出已发布的期；没开始写的领域显示「敬请期待」，让人看到整体蓝图。
- 数据全部来自文章 frontmatter 的 `domain` / `episode` / `date`，你照常发文章就会自动出现在这里。

### 十、 访问统计（看有多少人来、爱看哪几期）

网站已经接入了 **Vercel Analytics**（隐私友好：不放 cookie、不追踪个人身份，符合站点的调性）。它能告诉你访问量、最受欢迎的页面等数据——对一个长期积累的项目，这是判断「哪一期最被需要」的依据。

- **本地开发不会上报**，只有部署到线上才统计，不用担心污染数据。
- **首次需要在 Vercel 手动开一下**：打开 [Vercel](https://vercel.com) → 你的项目 → 顶部 **Analytics** 标签 → 点 **Enable**。开启后等有人访问，过一会儿就能看到图表了。
- 之后完全自动，你不用做任何事。

### 十一、 浏览量与点赞（每个作品都有，需一次性开通免费数据库）

每篇文章、每个图集/视频/音频的详情页，都会显示 **👁 浏览量** 和一个 **♥ 点赞按钮**：

- 浏览量：同一访客一次访问只 +1（不刷量）。
- 点赞：同一浏览器只能赞一次，可再点一下取消。
- 这些数字是**全站所有访客共享、永久累计**的。
- **列表里也会显示**：首页「最新更新」、`/lighthouse` 总览、各板块列表页里的每一项，都会带上 👁 浏览量和 ♥ 点赞数（只读，不可点）。

**为什么要开通数据库？** 你的网站是静态的，存不住"全站共享的数字"，需要一个**免费的云数据库**来存。这是一次性设置，约 2 分钟：

1. 打开 [Vercel](https://vercel.com) → 你的项目 → 顶部 **Storage**（存储）标签 → **Create Database** → 选 **Upstash**（Redis / KV）→ 按引导创建（选免费档即可）。
2. 创建时把它**连接（Connect）到你这个项目**——Vercel 会自动把数据库密钥写进项目的环境变量，你不用手动复制。
3. 到 **Deployments** → 最近一次部署点 **Redeploy** 重新部署一次。
4. 部署完成后，打开任意一篇文章或作品页，标题下方就会出现浏览量和点赞按钮了。

> **没开通之前会怎样？** 这个区域会**自动隐藏**，网站其它部分完全正常，所以你也可以以后再弄。本地开发（`npm run dev`）默认也看不到它，属正常现象。

### 十二、 文章评论（Giscus，基于 GitHub，免费无数据库）

文章底部可以开评论区，用的是 **Giscus**——评论存到你 GitHub 仓库的 Discussions 里，**免费、无需数据库**，读者用 GitHub 账号登录留言。一次性设置：

1. 打开你的 GitHub 仓库 `xiadong727/myweb` → **Settings** → 往下找到 **Features** → 勾选 **Discussions**。
2. 打开 [giscus.app](https://giscus.app)，在「仓库」里填 `xiadong727/myweb`；映射方式选 **pathname**；分类（Category）选 **Announcements**。
3. 页面下方「启用 giscus」会生成一段代码，里面有两个值：**`data-repo-id`** 和 **`data-category-id`**。
4. 把这两个值填到 Vercel：**Settings → Environment Variables**，新增：
   - `NEXT_PUBLIC_GISCUS_REPO_ID` = 上面的 repo-id
   - `NEXT_PUBLIC_GISCUS_CATEGORY_ID` = 上面的 category-id
5. **Redeploy** 一次。之后每篇文章底部就会出现评论区。

> 没配置之前评论区**自动隐藏**，不影响其它功能。
>
> **注意**：Giscus 评论**必须用 GitHub 账号登录后才能发表**，不支持匿名评论；而且 GitHub 在国内访问不稳定。如果你更看重「访客免登录、可匿名评论」，需要换成 Waline / Twikoo 这类方案（需要一个小后端）——可日后再议。

### 十三、 可视化后台：填表单发内容（不用手写文件）

如果你不想手动改 `.md` 和 JSON，可以用内置的**可视化后台**，填表单就能发文章/图片/视频/音频。

> ⚠️ **后台只在本地用**：因为线上服务器是只读的、也出于安全，后台**只在你本机 `npm run dev` 时可用**。流程是：本地填表 → 它自动写好文件、挂好菜单 → 你再照常提交推送上线。

**怎么用：**
1. 终端运行 `npm run dev`，浏览器打开 **`http://localhost:3000/admin`**。
2. 顶部选板块：**文章 / 图片 / 视频 / 音频**。
3. 填表单：
   - 文章：标题、文件路径、日期、标签、摘要、正文（可直接选图片**上传插图**，会自动存到 `public/images` 并把 Markdown 追加到正文）；若是「与光同行」主线，填上**领域 / 期号 / 金句**即可（会自动生成一鱼三吃关联键）。
   - 图片/视频/音频：填对应字段，图片可直接上传。
4. 选「**挂到哪个菜单分类**」（下拉是你现有的目录结构），点 **保存并写入文件**。
5. 后台会提示已生成的文件路径——这时去 Cursor 左侧「源代码管理」**提交并推送**即可上线。

> 它做的就是你平时手动做的两件事（写文件 + 改 `navigation.json`），只是变成了填表单、且不会填错格式。线上 `/admin` 打开会提示"仅本地可用"，属正常。

## 最后一步：如何发布到网上？（同步到 GitHub）

当你在本地做完了所有的增加、删除、修改，并且按 `Ctrl+S` (或 `Cmd+S`) 保存了文件后，你需要把它们推送到网上。

**只需三步：**
1. **打开源代码管理**：点击 Cursor 最左侧边栏的第三个图标（长得像树枝分叉的图标，叫 Source Control）。
2. **写更新说明**：在顶部的输入框（Message）里写一句话，比如：“新增了一篇日记和两张猫咪图片”。
3. **提交并推送**：
   * 点击输入框下方的 **Commit（提交）** 按钮。
   * 提交完成后，按钮会变成 **Sync Changes（同步更改）**，再点击它一下。

代码同步到 GitHub 后，Vercel 会在后台全自动为你更新网站。你只需要等待 1~2 分钟，刷新你的手机或电脑浏览器，就能看到最新内容了！
