# 个人网站搭建教程及指南

> 这份文档复盘了「拾光共长」这个个人网站是怎么搭起来的，并整理成一份**可复制的搭建指南**。如果你也想做一个能放文章、图片、视频、音频，可长期更新、还能被搜索引擎收录的个人网站，照着走即可。
>
> 全程**零服务器、零运维、基本零成本**（用免费额度），代码托管在 GitHub，部署在 Vercel，改完一键同步就自动上线。

---

## 一、先想清楚：这是一个什么样的网站？

它是一个**以内容为核心的个人站**，特点：

- **内容即文件**：文章是 Markdown，媒体（图片/视频/音频）和导航是 JSON，全部存在代码仓库里——没有后台、没有数据库也能跑。
- **静态为主、按需动态**：绝大多数页面在构建时生成（快、稳、利于 SEO）；只有「浏览量/点赞」「分享图」等少数功能用到一点点动态能力。
- **可长期演化**：从「几篇文章」长到「几百篇」都不用换架构。

适合：写作者、创作者、想做个人品牌或知识沉淀的人。不适合：需要用户注册体系、电商交易、复杂后台的场景。

## 二、技术选型（以及为什么）

| 用途 | 选型 | 为什么 |
| --- | --- | --- |
| 框架 | **Next.js（App Router）** | 同时支持静态生成与少量动态接口，生态成熟，部署到 Vercel 零配置 |
| 语言 | **TypeScript** | 类型安全，重构不慌 |
| UI | **React 19** | Next 默认 |
| 样式 | **Tailwind CSS** | 原子类，改样式快，主题切换方便 |
| 文章 | **Markdown + gray-matter** | 纯文本写作，`gray-matter` 解析头部信息（frontmatter） |
| 渲染 | **react-markdown + remark-gfm** | 把 Markdown 渲染成 React，支持表格/任务列表等 |
| 搜索 | **MiniSearch** | 纯前端全文搜索，构建时生成索引，无需搜索服务 |
| 图标 | **lucide-react** | 简洁的开源图标库 |
| 计数（浏览/点赞） | **Upstash Redis** | Serverless Redis，有免费额度，Vercel 一键集成 |
| 评论 | **Giscus** | 基于 GitHub Discussions，免费、无后端 |
| 访问统计 | **Vercel Analytics** | 隐私友好、一行接入 |
| 托管/部署 | **GitHub + Vercel** | push 即自动部署，免费额度足够个人站 |

> 核心思想：**能用静态解决的绝不上服务器，能用免费托管的绝不自己运维。**

## 三、整体架构

```
内容（你写的东西）
  content/articles/**.md      ← 文章（Markdown + frontmatter）
  data/galleries.json         ← 图片相册
  data/videos.json            ← 视频
  data/audios.json            ← 音频
  data/navigation.json        ← 左侧导航树（决定菜单结构）
        │
        ▼
构建时（Vercel 上 npm run build）
  · 读取所有 .md / .json
  · 生成每篇文章/每个作品的静态页面
  · 生成全文搜索索引 search-index.json
  · 生成 sitemap.xml / robots.txt / feed.xml(RSS)
        │
        ▼
运行时（访客打开网站）
  · 绝大多数是静态页面，CDN 直出，很快
  · 少量动态接口： /api/metrics（浏览量/点赞）、/api/og（分享图）
  · 客户端组件： 搜索弹窗、主题切换、目录高亮、数字动画等
```

一句话：**写文件 → 推送到 GitHub → Vercel 自动构建并上线。**

## 五、从零搭建步骤

### 1. 创建项目

```bash
npx create-next-app@latest my-site
# 选择： TypeScript ✓  App Router ✓  Tailwind CSS ✓  src/ 目录 ✓
cd my-site
npm run dev   # 打开 http://localhost:3000 看到初始页面
```

装上要用到的依赖：

```bash
npm i gray-matter react-markdown remark-gfm minisearch lucide-react
# 可选（增强功能，后面用到再装）：
npm i @upstash/redis @vercel/analytics
```

### 2. 规划目录结构

```
my-site/
├─ content/articles/        # 文章 .md（可建子文件夹分类）
├─ data/                    # galleries/videos/audios/navigation.json
├─ public/images/           # 全站公用图片
├─ scripts/                 # 构建脚本（生成搜索索引）
└─ src/
   ├─ app/                  # 路由与页面（App Router）
   │  ├─ (site)/            # 带侧边栏布局的页面组
   │  │  ├─ page.tsx        # 首页
   │  │  ├─ articles/[...slug]/page.tsx
   │  │  ├─ images / videos / audios / ...
   │  ├─ api/metrics/route.ts   # 浏览量/点赞接口（可选）
   │  ├─ sitemap.ts / robots.ts / feed.xml/route.ts
   │  └─ layout.tsx
   ├─ components/           # 复用组件
   └─ lib/                  # 读取内容、解析的工具函数
```

### 3. 内容模型：用 Markdown 写文章

每篇文章是一个 `.md` 文件，开头用 **frontmatter** 写元信息：

```markdown
---
title: "我的第一篇文章"
date: "2026-01-01"
excerpt: "一句话摘要，显示在列表和搜索结果里"
tags: ["随笔"]
---

正文从这里开始……
```

用 `gray-matter` 解析（`src/lib/articles.ts` 思路）：

```ts
import fs from "fs";
import path from "path";
import matter from "gray-matter";

const DIR = path.join(process.cwd(), "content/articles");

export function getArticleBySlug(slug: string) {
  const raw = fs.readFileSync(path.join(DIR, `${slug}.md`), "utf8");
  const { data, content } = matter(raw); // data=frontmatter, content=正文
  return { meta: data, content };
}
```

> 关键约定：**`slug`** = 文件相对 `content/articles/` 的路径（不带 `.md`）。它既是 URL，也是导航里引用文章的钥匙。

### 4. 媒体与导航：用 JSON

图片/视频/音频不写 Markdown，用 JSON 数组描述，例如 `data/videos.json`：

```json
[
  { "slug": "talks/intro", "title": "频道介绍", "kind": "embed",
    "embedUrl": "https://player.bilibili.com/player.html?..." }
]
```

**导航树**是整个网站的骨架 `data/navigation.json`，用嵌套 `children` 表达多级菜单；带 `slug` 的是文章/作品，带 `children` 的是分类：

```json
{
  "articles": { "nodes": [
    { "id": "cat-essay", "title": "随笔", "children": [
      { "id": "a1", "title": "我的第一篇文章", "slug": "my-first" }
    ]}
  ]}
}
```

页面渲染时递归读取这棵树，生成左侧可展开/折叠的目录。

### 5. 渲染文章页

App Router 用动态路由 `src/app/(site)/articles/[...slug]/page.tsx`：

```tsx
import ReactMarkdown from "react-markdown";
import remarkGfm from "remark-gfm";

export function generateStaticParams() {
  // 返回所有文章 slug，让 Next 在构建时把每篇都预渲染成静态页
}

export default async function Page({ params }) {
  const { slug } = await params;
  const article = getArticleBySlug(slug.join("/"));
  return (
    <article>
      <h1>{article.meta.title}</h1>
      <ReactMarkdown remarkPlugins={[remarkGfm]}>{article.content}</ReactMarkdown>
    </article>
  );
}
```

阅读体验增强（都是可选的小组件）：文章目录（从正文标题提取）、阅读进度条、预计阅读时长、上一篇/下一篇。

### 6. 全文搜索（构建时生成索引）

写一个脚本 `scripts/build-search-index.mjs`：读取所有文章，抽取标题/摘要/正文，用 MiniSearch 建索引，输出到 `public/search-index.json`。在 `package.json` 里挂到构建前：

```json
"scripts": {
  "predev": "node scripts/build-search-index.mjs",
  "prebuild": "node scripts/build-search-index.mjs",
  "dev": "next dev",
  "build": "next build"
}
```

前端搜索框 `fetch('/search-index.json')` 后用 MiniSearch 在浏览器里查，**零搜索服务、零成本**。

### 7. 样式与主题

Tailwind 写样式；主题用 **CSS 变量**实现：在 `globals.css` 里为每套主题定义一组变量（`--background`、`--primary` 等），切换主题就是换 `data-theme`。这样可以轻松做「暖色 / 深色 / 多套配色」切换。

### 8. SEO 三件套（让别人搜得到）

Next 提供约定式文件，放在 `src/app/` 下即可自动生成：

- `sitemap.ts` → 站点地图 `sitemap.xml`
- `robots.ts` → `robots.txt`
- `feed.xml/route.ts` → RSS 订阅源

再给每篇文章加 `generateMetadata` 输出标题、描述、Open Graph 信息，分享到社交平台时就有好看的预览。

## 六、部署上线（GitHub + Vercel）

1. 在 GitHub 新建一个仓库，把项目推上去：
   ```bash
   git init && git add -A && git commit -m "init"
   git branch -M main
   git remote add origin https://github.com/你的用户名/你的仓库.git
   git push -u origin main
   ```
2. 打开 Vercel → **Add New Project** → 选中这个 GitHub 仓库 → **Deploy**。Vercel 会自动识别 Next.js，构建并分配一个 `xxx.vercel.app` 网址。
3. 以后**每次 `git push`，Vercel 自动重新部署**，1～2 分钟就更新上线。

**绑定自己的域名**（可选）：Vercel → 项目 → Settings → Domains，添加你买的域名并按提示配置 DNS。绑定后建议设一个环境变量 `NEXT_PUBLIC_SITE_URL = https://你的域名`，让 sitemap / RSS / 分享图用到正确的绝对网址。

## 七、可选增强功能

这些都不是必须的，按需添加，**不开通就自动隐藏**，不影响主站：

- **浏览量 / 点赞**：用 Upstash Redis（Vercel Storage 一键创建，选 Pay As You Go 免费档）。写一个 `/api/metrics` 接口做 `incr`（自增）与读取，前端用 localStorage 去重防刷。
- **评论**：Giscus（基于 GitHub Discussions），在 [giscus.app](https://giscus.app) 生成配置，填两个 ID 即可。**注意读者需用 GitHub 登录**；要免登录/匿名评论得换 Waline、Twikoo 等（需自带小后端）。
- **访问统计**：`@vercel/analytics`，在根布局放一个 `<Analytics />`，再去 Vercel 的 Analytics 标签点 Enable。隐私友好、无 cookie。
- **分享配图（OG 图）**：用 `next/og` 的 `ImageResponse` 动态生成一张品牌化预览图（标题 + Logo），分享到微信/小红书更好看。
- **数字动画 / 灯箱 / 标签聚合页 / 404 页**：纯前端小组件，锦上添花。

## 八、国内访问优化（重要提醒）

Vercel 在**中国大陆没有节点**，且部分 IP 会被限速，国内访问偏慢。这是托管层问题，不是代码 bug。能做的：

1. **代码层**（已能做到）：第三方资源懒加载（视频 iframe 点击再加载、评论滚动到再加载）、图片懒加载、减少首屏请求。
2. **数据库就近**：Upstash 数据库选 **新加坡 / 东京** 区域，别选美国，点赞/浏览量响应更快。
3. **托管层（治本，需取舍）**：
   - 维持 Vercel：最省心，但国内慢；
   - 换**香港服务器**：明显更快、**免备案**，但要会部署 Node；
   - **大陆服务器 + ICP 备案**：最快，但要买国内服务器并完成备案（备案必须有大陆服务器，没有服务器无法备案）。

## 九、日常维护（上线后）

写新内容的固定动作就三步：

1. **写文件**：在 `content/articles/` 加 `.md`（或在对应 JSON 里加一条媒体）。
2. **挂菜单**：在 `data/navigation.json` 里加一条，`slug` 对上文件。
3. **同步**：编辑器里 Commit + Push（或 `git push`）。Vercel 自动构建上线。

> 全程不碰服务器、不碰数据库。内容就是文件，版本历史都在 Git 里，删错了也能找回。

## 十一、常见坑

- **改了文件网页没变**：本地确认存盘；线上确认已 `git push` 且 Vercel 部署成功。
- **导航里点不开文章**：多半是 `navigation.json` 里的 `slug` 和文件路径对不上（差一个字、多了 `.md`、忘了子目录）。
- **JSON 报错**：少了逗号、多了逗号、引号是中文引号——用编辑器的 JSON 校验。
- **浏览量/评论不显示**：对应的数据库/Giscus 没配或没 Redeploy；本地开发本来就不显示，属正常。
- **构建失败**：看 Vercel 的构建日志，通常是某个文件语法错误或字段缺失。

## 结语

这套方案的精髓是：**把网站当成「一堆文件 + 一次自动化构建」**。你专注写内容，剩下的交给 Git 和 Vercel。它足够简单（新手能维护），又足够能打（搜索、SEO、互动、统计一样不少），并且可以陪你的内容一起长很多年。

祝你也搭出一个属于自己的、能长期生长的小站。🌱
