## 一、现在有各大平台了，我为什么还要做一个个人网站？

公众号、知乎、小红书、B 站……平台又多又香，为什么还要折腾一个自己的站？我的答案是：**平台是租来的房子，网站才是自己的家。**

具体来说有这么几条：

1. **数据和内容真正属于自己。** 在平台上，账号、内容、粉丝的"所有权"其实都不完全在你手里——规则一变、账号一封，多年积累可能一夜清零。自己的网站，内容是本地的一堆文件，我想搬到哪就搬到哪。
2. **不被算法和流量绑架。** 平台希望你迎合推荐机制：标题要炸、要追热点、要日更。而我想做的是**能穿越周期、可以慢慢沉淀**的东西，不想被"今天涨了多少粉"牵着走。
3. **完全自由的表达和排版。** 想怎么排版就怎么排，想加什么功能就加什么功能，没有平台的样式限制和审核口径。
4. **长期主义的载体。** 我给「拾光共长」定的愿景是"用十年时间，把人类智慧打磨成每个家庭都能用的思想工具"。这种十年尺度的事，需要一个**自己能掌控、能持续迭代**的地方。
5. **一个统一的"数字自留地"。** 文章、图片、视频、音频，散落在各个平台很难聚合；在自己站里，它们可以围绕同一套领域体系组织起来，形成一个整体。

> 一句话：**平台用来获取流量，网站用来沉淀资产。** 两者不冲突，我把平台当"分发渠道"，把网站当"老家"。

## 二、个人网站的技术选型思考

选型时我给自己定了几条原则：**长期可维护、内容与代码分离、几乎零运维成本、未来能自由扩展。**

| 维度 | 我的选择 | 为什么 |
|---|---|---|
| 框架 | **Next.js（App Router）+ React + TypeScript** | 既能静态生成（快、利于 SEO），又保留了需要时上动态能力的余地；生态成熟 |
| 样式 | **Tailwind CSS** | 写起来快、风格统一，主题色/暗色模式好控制 |
| 内容存储 | **Markdown + 少量 JSON**，不上数据库 | 内容即文件，跟着 Git 走，天然有版本历史、天然可备份、迁移零成本 |
| 部署 | **GitHub → Vercel 自动部署** | push 即上线，免费额度对个人站绰绰有余，省去服务器运维 |
| 浏览量/点赞 | **Upstash（Serverless Redis）** | 唯一需要"动态"的地方，用按量计费的云 Redis，平时几乎不花钱 |

几个**有意识的取舍**：

- **为什么不用 WordPress / 现成博客平台？** 它们上手快，但要么要自己运维服务器和数据库，要么又回到"被平台框住"的老路。我想要的是完全可控、可编程。
- **为什么内容不进数据库？** 个人站的内容量不大，用 Markdown 文件反而更轻：编辑器随便选、版本管理交给 Git、备份就是复制文件夹，**没有数据库就没有数据库的运维和丢失风险**。
- **为什么是 Vercel？** 对个人项目几乎零成本、零运维，自动 HTTPS、自动构建。（唯一的遗憾是国内访问偏慢，这点放在最后的规划里说。）

## 三、具体实现方案

整体架构其实很简单，可以一句话概括：**内容是文件，代码把文件渲染成网站，平台自动部署。**

```
            ┌─────────────────────────────┐
  写内容 →  │  Markdown 文章 / JSON 配置    │   ← 也可用「本地后台」可视化编辑
            │  图片等媒体放 public/images   │
            └──────────────┬──────────────┘
                           │  git push
                           ▼
            ┌─────────────────────────────┐
            │   GitHub（版本管理 + 备份）   │
            └──────────────┬──────────────┘
                           │  自动触发
                           ▼
            ┌─────────────────────────────┐
            │  Vercel：构建 Next.js → 上线  │
            │  浏览量/点赞 → Upstash Redis  │
            └─────────────────────────────┘
```

*（上面这张是文字版架构图，你也可以画一张更好看的放到 `cogrow-arch.png`。）*

在这个骨架之上，我做了几个自己比较满意的功能：

**1. 「与光同行 · 10 大领域」内容体系。**
把内容按哲学、思维、品格、历史、科学、艺术、人际、财富、生活、心灵 10 个领域组织，每个领域可以同时容纳文章、图片、视频、音频，首页和栏目页会自动统计各领域的作品数与进度。

![与光同行栏目](/images/cogrow-lighthouse.png)
*每个领域的作品数会自动汇总，点进去能看到该领域下分类的全部作品。*

**2. 一份内容、多种形态（"一鱼三吃"）。**
同一个主题可以拆成文章 + 视频 + 音频，用一个"期号"串起来，方便交叉跳转。

**3. 一个本地可视化后台（CMS）。**
虽然内容是 Markdown 文件，但我做了一个**只在本地运行**的后台，可以可视化地新建、编辑、移动内容，调样式，最后一键"发布"（其实就是帮我 commit + push）。线上环境这个后台是**完全关闭**的，保证安全。

![本地后台](/images/cogrow-admin.png)
*本地后台「管理内容」：可视化管理所有文章 / 图片 / 视频 / 音频。*

**4. 微信 / B 站内容一键导入。**
贴一个公众号文章链接或 B 站视频链接，自动抓取正文、把图片重新存到本地（解决防盗链）、生成 Markdown，省去大量搬运的体力活。

![一键导入](/images/cogrow-import.png)
*把链接粘进去，自动转成站内内容。*

**5. 一些体验细节。**
全站搜索、明暗主题、移动端适配、文末金句卡、浏览量与点赞统计……这些"小事"加起来，才让它从"能用"变成"想用"。

## 四、花了多长时间

> 👉 下面是我的大致节奏，你按自己的真实情况改成具体数字就行：

- **搭骨架（框架 + 部署跑通）**：大约 X 天，第一次 `push` 看到线上能访问的那一刻最有成就感。
- **核心功能（内容体系 + 后台 + 导入）**：陆续利用业余时间，大约 X 周。
- **打磨细节（移动端、配色、各种小优化）**：这部分是"无底洞"，但也是最享受的——基本是**边用边改**，持续进行中。

总体感受：**先用最小可用版本上线，再慢慢迭代**，比憋一个"完美版本"再发布要爽得多，也更容易坚持下去。

## 五、成本

这可能是最让人意外的部分——**目前几乎是零成本**：

| 项目 | 费用 |
|---|---|
| 域名（cogrow.top） | 约 ¥几十 / 年 |
| Vercel 托管 | **¥0**（个人免费额度足够） |
| Upstash（浏览量/点赞） | **≈¥0**（按量，流量小基本不花钱） |
| GitHub | **¥0** |
| **合计** | **≈ 一顿饭钱 / 年** |

唯一明显的代价不是钱，而是**国内访问速度**：Vercel 的节点在境外，国内打开偏慢。如果以后要为国内访问提速，需要做 ICP 备案 + 国内服务器 / CDN，那时成本大约会到 **¥100–300 / 年**。这笔账我觉得很划算——用一顿饭钱，换一个完全属于自己的、能用十年的地方。

## 六、未来想法和规划

这个站对我来说才刚刚开始，接下来想做的事还有很多：

- **国内访问提速**：完成备案，上国内服务器或 CDN，让国内的朋友也能秒开。
- **评论 / 互动**：加一套评论系统，让"记录"变成"交流"。
- **RSS 与开放**：提供订阅，不依赖任何平台也能持续触达读者。
- **内容自动化**：让"多平台 → 网站"的同步更顺，减少搬运。
- **把「与光同行」真正做深**：这是核心——十年时间，一个领域一个领域地，把人类的智慧打磨成每个家庭都能读懂、用得上的"思想工具"。

> 做平台是为了被更多人看见，做网站是为了对自己负责。
> 如果你也有想长期沉淀的东西，**真心推荐你也给自己造一个"数字自留地"**——它不一定要多复杂，但它会一直在那里，陪你慢慢长大。
