# 个人网站搭建教程及指南（白话版）

> 这份是给「完全不懂技术的小白」看的版本。我尽量不用术语，用大白话和打比方，带你从零拥有一个属于自己的网站——可以放文章、照片、视频、音频，能一直更新，还能被百度/谷歌搜到。
>
> 别怕，**全程几乎不用写代码，也不用花钱**。跟着点就行。

---

## 一、先用大白话讲清楚：这玩意是怎么回事？

你可以把这个网站想象成一个**活页夹**：

- 你写的每篇文章 = 一张**活页纸**，放进活页夹里。
- 照片、视频、音频 = 另外几种活页纸。
- 左边的目录菜单 = 活页夹的**分隔标签**，告诉别人东西放在哪一格。

而这些「活页纸」其实就是你电脑里的一个个**文件**。你只要：

1. 在电脑上**写好/放好文件**，
2. 点一下「上传」，
3. 网站就自动更新了。

没有复杂后台，没有数据库，不用懂服务器。**内容就是文件，文件就是内容。**

---

## 二、你需要准备的 4 样东西（都免费）

| 东西 | 是什么（大白话） | 去哪弄 |
| --- | --- | --- |
| **Cursor**（或 VS Code） | 一个写东西、传文件的「工作台」软件 | <https://cursor.com> |
| **Node.js** | 让网站能在你电脑上「预览」的小引擎 | <https://nodejs.org>（下载推荐版本，一路点下一步） |
| **GitHub 账号** | 你文件的「云端保险柜」，存所有内容和历史 | <https://github.com> |
| **Vercel 账号** | 一个「自动印刷厂」：你一上传，它就把网站印出来给全世界看 | <https://vercel.com>（用 GitHub 登录） |

打个比方：
- **GitHub** 像你家的**云端储物间**，东西都存这儿，还能记住每次的改动（删错了能找回）。
- **Vercel** 像一家 **24 小时自动印刷厂**，盯着你的储物间，你一放新东西，它立刻重新「印刷」出网站。

## 三、把网站「立起来」（一次性，约 30 分钟）

> 这一步是搭骨架。如果你是接手别人已经做好的项目（比如有人给了你一个现成的网站文件夹），可以**跳到第四步**直接开始写内容。

### 1. 拿到网站的「初始模板」
最省事的办法：让懂一点的朋友（或 AI 助手）帮你用 **Next.js** 生成一个初始项目，并放进一个文件夹。你会得到一个装满文件的文件夹——这就是你的网站「毛坯房」。

### 2. 在 Cursor 里打开这个文件夹
打开 Cursor → 「打开文件夹」→ 选中它。左边能看到一堆文件，别慌，你常用的就那几个。

### 3. 本地预览（看看长啥样）
在 Cursor 底部打开「终端」（菜单里找 Terminal），输入这行回车：

```
npm run dev
```

然后浏览器打开 `http://localhost:3000`，就能看到你的网站了。**这个只有你自己电脑能看**，是用来预览的。改完文件按保存，网页会自动刷新。

### 4. 传到云端 + 上线
- 先在 **GitHub** 新建一个仓库（储物间），把项目传上去。
- 再到 **Vercel** 点「新建项目」，选中这个 GitHub 仓库，点 **Deploy（部署）**。
- 等 1～2 分钟，Vercel 会给你一个网址（像 `你的名字.vercel.app`），**这就是全世界能访问的网站了！**

以后你**每次上传改动，网站都会自动更新**，不用再操作 Vercel。

## 四、最常做的事：发一篇新文章（3 步）

这是你以后 99% 的时间在做的事，记住这三步就够了：

### 第 1 步：写文章
在 `content/articles/` 文件夹里新建一个 `.md` 文件（比如 `my-diary.md`）。`.md` 就是一种最简单的文本格式。文件开头照抄这几行（叫「头部信息」），再往下写正文：

```
---
title: "今天天气真好"
date: "2026-01-01"
excerpt: "一句话简介，会显示在列表里"
---

正文从这里开始，想写啥写啥……
```

> 小贴士：`title` 是标题，`date` 是日期，`excerpt` 是摘要。三个引号里的内容随便改。

### 第 2 步：把它放进菜单
光写文件，网站上还看不到，得去「目录表」里登记一下。打开 `data/navigation.json`，在合适的位置加一段：

```
{ "id": "my-diary", "title": "今天天气真好", "slug": "my-diary" }
```

> `slug` 必须和你的文件名一样（不要带 `.md`）。记得跟前一行之间用逗号隔开。

### 第 3 步：上传（一键上线）
在 Cursor 左侧找到「源代码管理」（一个像树杈的图标）：
1. 写一句说明，比如「新增一篇日记」；
2. 点 **提交（Commit）**；
3. 再点 **同步 / 推送（Sync / Push）**。

搞定！等 1～2 分钟，刷新你的网站，新文章就在了。

## 五、放照片 / 视频 / 音频

和发文章很像，也是「**填一条信息 + 挂到菜单**」两步，只是信息填在不同的文件里：

- **照片**：图片文件放进 `public/images/`，再在 `data/galleries.json` 里加一条。
- **视频**：建议先传到 B 站，拿到嵌入链接，填进 `data/videos.json`。
- **音频**：把音频链接填进 `data/audios.json`。

填完同样去 `data/navigation.json` 挂到菜单里。具体字段照着文件里已有的样板改最快。

> 记住口诀：**先在数据文件里填内容，再去菜单文件里挂位置，两边用 `slug` 对上号。**

## 六、网站自带的「贴心功能」（不用你操心）

这些都是自动的，你正常发内容就有：

- 🔎 **搜索**：右上角能搜全站文章。
- 📂 **左侧目录**：自动折叠展开。
- 📊 **首页数据墙**：自动统计你有多少篇文章、多少张图、总浏览量等，还有数字滚动动画。
- 👁 **浏览量 + ♥ 点赞**：每篇文章都有（需要开通一个免费数据库，见下）。
- 🔗 **分享好看**：把文章链接发微信/朋友圈，会自动带一张漂亮的预览图。
- 🌙 **多种主题**：右上角能换配色（含深色模式）。
- 🗺️ **被搜索引擎收录**：自动生成站点地图，利于百度/谷歌找到你。

## 七、想要「浏览量、点赞、评论」？（可选，各 2 分钟）

这几个需要简单开通一下，**不开也完全不影响网站**，它们会自动隐藏。

### 浏览量 / 点赞
网站本身存不住「全站共享的数字」，需要一个**免费的云数据库**：
1. 打开 Vercel → 你的项目 → 顶部 **Storage** → **Create Database** → 选 **Upstash**；
2. 套餐选 **Pay As You Go**（这就是免费档，用量小基本不花钱）；地区选**新加坡或东京**（离国内近、快）；
3. 创建时连接到你的项目，然后去 **Deployments** 点一次 **Redeploy**。
4. 完成后，每篇文章就有浏览量和点赞了。

### 评论
用的是 **Giscus**，评论存在你的 GitHub 里，免费。但有一点要知道：

> ⚠️ **读者必须登录 GitHub 账号才能评论，不能匿名。** 而且 GitHub 在国内有时打不开。如果你希望「访客不用登录就能评论」，那需要换成别的评论系统（比如 Waline），稍微麻烦些，可以以后再说。

## 八、关于「国内打开慢」这件事

实话实说：我们用的 Vercel「印刷厂」在国外，**国内访问会偏慢**，这不是你做错了什么。几个选择：

1. **先这样用**：最省心，能打开，就是有点慢。
2. **换香港的服务器**：会快不少，而且**不用备案**，但需要懂一点部署。
3. **用国内服务器 + 备案**：最快，但要买国内服务器并办「ICP 备案」（备案必须先有大陆服务器，没有服务器就办不了）。

新手建议：**先用着，等内容多了、真有人看了，再考虑提速。**

## 九、花多少钱？

- 写文章、传文件、上线：**0 元**。
- 想要自己的域名（比如 `mysite.com`）：**约一年几十块**。
- 想要国内访问飞快（服务器+备案）：**约一个月几十块**。

也就是说：**最简单的玩法，一分钱不花。**

## 十、遇到问题别慌（自查清单）

- **改了没反应？** → 确认存盘了；确认点了「同步/推送」；去 Vercel 看看是不是还在部署。
- **文章点不开？** → 多半是菜单文件 `navigation.json` 里的 `slug` 和文件名没对上（差个字、多了 `.md`、忘了文件夹）。
- **保存时报错红字？** → 多半是 JSON 文件里少了或多了**逗号**，或者引号用成了**中文引号** `"`（要用英文 `"`）。
- **浏览量/评论不见？** → 没开通对应功能，或没重新部署；本地预览本来就不显示，正常。
- **彻底卡住了？** → 把报错截图发给懂的人或 AI，描述清楚「我做了什么、出现了什么」，通常很快能解决。
